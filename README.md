Приложение написано с учетом Clean Architecture и паттерна MVVM
А именно деление приложения на три слоя Domain, Data, Presentation
- Четкое разделение ответственностей
- Независимость слоев и легкая замена компонентов при необходимости

Получение данных идет засчет ViewModel и хранящихся там потоков StateFlow

Для решения задачи с хранением локального статуса задачи (Task) была использована библиотека Room DB
Алгоритм решения:
- Данные берутся с сервера, если в локальной БД пусто то локальный статус = статусу с сервера
- При смене локального статуса появляется новая запись в БД с task-id и status
- При повторном запросе с сервера, учитываются записи с БД и в случае чего локальный статус меняется вне зависимости от статуса сервера
